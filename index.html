Section 01
1 Undetermined Loss คืออะไร?

ทำความเข้าใจนิยาม ที่มา และความสำคัญของ UDL ในอุตสาหกรรมน้ำตาล

สูตรคำนวณ
UDL = Total Losses − Bagasse Loss − Filter Cake Loss − Final Molasses Loss
Total Losses = 100% − Overall Recovery (%)
ทั้ง Bagasse, Filter Cake, Final Molasses เป็น "Determined Loss" ที่วัดได้จากห้อง Lab

Total Losses — การสูญเสียรวม

คือส่วนต่างระหว่างน้ำตาลที่อยู่ในอ้อย (Pol in Cane) กับน้ำตาลที่ผลิตได้จริง คิดเป็น % ของ Pol in Cane

Determined Losses — สูญเสียที่วัดได้

  • Bagasse Loss: Pol ที่ตกค้างในชานอ้อย (~3%)
  • Filter Cake Loss: Pol ที่สูญเสียไปกับกากหม้อกรอง (~0.3%)
  • Molasses Loss: Pol ที่ไม่สามารถตกผลึกได้ ออกไปกับกากน้ำตาล (~8-10%)

Undetermined Loss — สูญเสียที่ไม่ทราบสาเหตุ

คือส่วนที่เหลือหลังหักการสูญเสียที่วัดได้ทั้ง 3 รายการออกแล้ว — เป็น "ช่องว่าง" ที่ไม่สามารถระบุได้ว่าน้ำตาลหายไปไหน

UDL ที่ดีควร < 4% | ค่าเฉลี่ย MLS 12 ฤดู = 6.38%

ทำไม UDL ถึงสำคัญ?

  • UDL ทุกๆ 1% ที่ลดลง = น้ำตาลเพิ่ม ~700-1,000 ตัน/ฤดู
  • เป็นตัวชี้วัดประสิทธิภาพโดยรวมของโรงงาน
  • อาจเกิดจากการสูญเสียจริง หรือความคลาดเคลื่อนในการวัด
  • ต้องวิเคราะห์ทั้ง Physical Loss + Analytical Error
Section 02
2 กระบวนการผลิตน้ำตาล

ตั้งแต่อ้อยเข้าโรงงานจนได้น้ำตาล — แต่ละขั้นตอนมี Yield Loss, ความเสี่ยง และโอกาสลด UDL
สรุปจากเอกสาร WI ทั้ง 32 ฉบับ จาก 6 แผนก

1

รับอ้อย (Cane Receiving)

Weighing → Sampling → Quality Analysis
ขั้นตอนการทำงาน
  1. อ้อยรถบรรทุกเข้าชั่งน้ำหนักที่สะพานชั่ง
  2. สุ่มเก็บตัวอย่างอ้อยด้วยเครื่อง Core Sampler ให้เป็นตัวแทนของรถ/Lot
  3. หั่นตัวอย่าง วิเคราะห์ %Pol, %Fiber, %Brix, Purity ในห้อง Lab
  4. บันทึกผลเป็น Baseline สำหรับคำนวณ Sugar Balance ทั้งระบบ
  5. อ้อยถูกลำเลียงเข้า Cane Carrier เข้าสู่กระบวนการหีบ
แหล่งที่มาของ Loss
  • ไม่มี Physical Loss โดยตรง แต่ความคลาดเคลื่อนที่นี่ส่งผลต่อ UDL ทั้งระบบ
  • อ้อยไฟไหม้ทำให้เกิดการย่อยสลายน้ำตาล (Inversion) ก่อนเข้าโรงงาน
ความเสี่ยง
  • อ้อยไฟไหม้ (Burnt Cane) สูง → Dextran + Inversion → UDL สูง
  • สุ่มตัวอย่างไม่เป็นตัวแทน → ข้อมูลพื้นฐานคลาดเคลื่อน
  • เครื่องมือวิเคราะห์ไม่ Calibrate → Phantom UDL
โอกาสลด UDL
  • ลดอ้อยไฟไหม้ให้ต่ำกว่า 5% (ปัจจุบัน 25/26 = 5.0%)
  • เพิ่มความถี่ Calibration เครื่องวิเคราะห์ตาม ML-WI-3330-002
  • ตรวจสอบระบบสุ่มตัวอย่างให้เป็นตัวแทนที่ดี
อ้างอิง WI: ML-WI-1630-001 · ML-WI-3330-002

หีบอ้อย (Milling)

Preparation → 5 Mill Tandems → Mixed Juice | แผนก 3110
ขั้นตอนการทำงาน
  1. อ้อยผ่าน Leveller/Shredder เตรียมความพร้อม (เป้าหมาย PI 89-91)
  2. ป้อนอ้อยเข้าลูกหีบชุดที่ 1-5 ต่อเนื่องกัน (Tandem)
  3. ฉีดน้ำ Imbibition ที่ลูกหีบท้ายๆ (~30% น้ำหนักอ้อย, ≥75°C) เพื่อสกัดน้ำตาลจากเส้นใยเพิ่ม
  4. ควบคุมแรงดันไฮดรอลิคกดลูกหีบแต่ละชุด (2,000-3,300 PSIG)
  5. เก็บน้ำอ้อยรวม (Mixed Juice) ส่งต่อแผนกทำใส ส่วนกากอ้อย (Bagasse) ส่งเป็นเชื้อเพลิงหม้อไอน้ำ
  6. ตรวจ Brix Curve และ Pol%Bagasse เป็นระยะเพื่อยืนยันประสิทธิภาพ
แหล่งที่มาของ Loss — Bagasse Loss ~3.0-3.9%
  • Pol ที่ตกค้างในชานอ้อย = Determined Loss (วัดได้)
  • เป้าหมาย: Pol%Bag ≤ 1.7%, Extraction ≥ 97%
  • ลด Pol%Bag 0.10% = เพิ่มน้ำตาล 5.80 ตัน/วัน (ที่ 20,000 TCH)
ความเสี่ยง
  • PI ต่ำกว่า 89 → อ้อยไม่ถูกบดละเอียด → สกัดน้ำตาลได้น้อย
  • Roller สึก/เบี้ยว → การบีบไม่สม่ำเสมอ → Pol%Bag สูง
  • น้ำ Imbibition ต่ำกว่า 75°C → สกัดน้ำตาลจากเส้นใยได้ไม่ดี
  • น้ำอ้อยรั่วไหลระหว่าง Mill → สูญเสียโดยตรง (UDL)
  • หยุด-เริ่มเครื่องบ่อย → อ้อย/น้ำอ้อยค้างระบบนาน → เสี่ยง Inversion
โอกาสลด UDL
  • เพิ่ม PI ให้ถึง 89-91 ด้วยการ Set ชุด Preparation ให้เหมาะสม
  • ตรวจสอบรอยรั่วของ Juice Tray ทุกสัปดาห์
  • ควบคุมอุณหภูมิ Imbibition ≥ 75°C อย่างสม่ำเสมอ
  • ใช้สูตรคำนวณน้ำตาลสูญเสียจริง (ตัน/วัน) จาก ML-SP-3110-003 ผูกกับ Purity Drop ทุกกะ
อ้างอิง WI: ML-WI-3110-007 ถึง 011, ML-SP-3110-003

ทำใส (Clarification)

Liming → Heating → Flocculation → Settling → Filtration | แผนก 3210
ขั้นตอนการทำงาน
  1. น้ำอ้อยรวมเข้าเครื่องอุ่น (Heater) แรกจนถึง ~70-75°C
  2. เติมน้ำปูนขาว (Liming) ปรับ pH ให้เป็นกลาง-ด่างอ่อน (~7.0-7.2)
  3. อุ่นซ้ำและส่งเข้า Flash Tank เพื่อไล่ฟองอากาศ
  4. เติมสารพักใส (Flocculant) แล้วส่งเข้าถัง Clarifier ให้ตะกอนแยกตัว (Settling)
  5. น้ำอ้อยใส (Clear Juice) ส่วนบนไหลไปแผนกระเหย ส่วนโคลนด้านล่างส่งไปกรองที่ Vacuum Filter (RVF)
  6. กากตะกอนกรอง (Filter Cake) นำออกนอกระบบ ส่วนน้ำกรอง (Filtrate) หมุนเวียนกลับเข้ากระบวนการ
แหล่งที่มาของ Loss — Filter Cake Loss ~0.2-1.2%
  • Pol ในกากหม้อกรอง (Filter Cake) = Determined Loss
  • การรั่วไหลของน้ำอ้อย/Syrup = Direct UDL
  • pH ไม่เหมาะสม → Inversion (น้ำตาลซูโครสถูกย่อยสลาย) → UDL
ความเสี่ยง
  • pH ต่ำกว่า 6.8 → Inversion → สูญเสีย Sucrose ที่วัดไม่ได้ (UDL)
  • เครื่อง Heater รั่ว → น้ำอ้อยเจือปนน้ำ Condensate → Dilution Loss
  • Mud Recirculation ไม่ดี → Pol ใน Filter Cake สูง
  • น้ำ Syrup รั่วไหลลงท่อระบาย → สูญเสียโดยตรง
โอกาสลด UDL
  • ติดตั้ง Brix Monitor ที่ท่อน้ำทิ้ง/Condensate เพื่อตรวจจับการรั่ว
  • ควบคุม pH อย่างเข้มงวดด้วยระบบ Auto Liming
  • ปฏิบัติตาม ML-WI-3210-012 อย่างเคร่งครัด
  • ทดสอบ Tube ของ Heater ก่อนเปิดหีบทุกฤดู
อ้างอิง WI: ML-WI-3210-001 ถึง 005, 012

ระเหย (Evaporation)

Multiple-Effect Evaporation → Syrup (Brix 60-65°)
ขั้นตอนการทำงาน
  1. น้ำอ้อยใสเข้าระบบระเหยหลายเอฟเฟกต์ (Multiple-Effect Evaporator) ต่อเนื่องกัน 4-5 Effect
  2. ไอน้ำจากหม้อไอน้ำให้ความร้อน Effect แรก ไอที่เกิดขึ้นถูกส่งต่อเป็นแหล่งความร้อนของ Effect ถัดไป
  3. ตรวจสอบ Brix ของน้ำเชื่อมที่ออกแต่ละ Effect เพื่อติดตามอัตราการระเหย
  4. ตรวจ Brix ใน Condensate เป็นระยะ เพื่อยืนยันว่าไม่มี Tube รั่ว
  5. น้ำเชื่อม (Syrup) เข้มข้นถึง Brix 60-65° ถูกส่งต่อแผนกเคี่ยว
แหล่งที่มาของ Loss
  • ไม่มี Yield Loss โดยตรงในทางทฤษฎี (แค่ระเหยน้ำ)
  • แต่! Entrainment (น้ำอ้อยถูกพาไปกับไอน้ำ) = UDL
  • Calandria Tube รั่ว → Sugar เจือปนใน Condensate = UDL
ความเสี่ยง
  • Tube รั่ว → น้ำตาลหลุดเข้า Condensate → ถูกทิ้งทั้งหมด
  • Entrainment สูง → น้ำตาลถูกพาไปกับไอน้ำ
  • อุณหภูมิสูงเกิน + Retention Time นาน → Inversion
โอกาสลด UDL
  • ตรวจ Brix ใน Condensate ทุก 2 ชั่วโมง
  • ทำ Pressure Test ของ Calandria Tube ก่อนเปิดหีบ
  • ทำความสะอาด Evaporator ตามแผนเพื่อลด Retention Time

เคี่ยว/ต้มผลึก (Pan Boiling)

3-Boiling System: A/B/C Seed → A/B/C Massecuite | แผนก 3220
ขั้นตอนการทำงาน
  1. เตรียมหม้อเคี่ยว ทดสอบ Hold Vacuum ก่อนเริ่มต้ม (ลดลง >20 หน่วยใน 20 นาที = รั่ว)
  2. ต้ม A-Seed จากน้ำเชื่อม + B-Magma (45-50%) จนได้เกรนน้ำตาล 0.8-0.9mm
  3. ต้มต่อเป็น A-Massecuite จนได้ Brix/Purity Drop ตามเป้า แล้ว Discharge สู่ Crystallizer
  4. ทำซ้ำระบบเดียวกันสำหรับ B-Boiling (ฐาน C-Magma, เกรน 0.6-0.7mm) และ C-Boiling (ฐาน A/B-Molasses ด้วยเทคนิค Slurry Seed Shock 450-500ml)
  5. ควบคุม Boiling Profile ตาม Set Point 10 ขั้นที่ตั้งไว้ใน DCS + Micro-Polar Brix
  6. Massecuite ที่ต้มเสร็จแต่ละ Batch ส่งต่อแผนกปั่นแยก
แหล่งที่มาของ Loss
  • Purity Drop ต่ำ → น้ำตาลตกค้างใน Molasses มากขึ้น (Molasses Loss สูง)
  • Crystal Content ต่ำ → ประสิทธิภาพการตกผลึกต่ำ → Recovery ลดลง
  • น้ำตาลค้างใน Pan/ท่อ/Tank → Physical Loss (UDL)
  • Mass/Mol หกล้นระหว่างต้ม/ย้าย → ตักกลับไปละลายใหม่ แต่ไม่มีการชั่ง/บันทึกปริมาณ (Loss ที่มองไม่เห็นในตัวเลข)
ความเสี่ยง
  • False Grain → ผลึกไม่สม่ำเสมอ → แยก Centrifugal ยาก → Loss สูง
  • Pan รั่ว (Hold Vacuum ไม่ผ่าน) → อากาศเข้า → ผลึกไม่ดี
  • Tube Calandria รั่ว → Steam เข้าน้ำตาล → Dilution
  • Transfer ไม่สมบูรณ์ → น้ำตาลค้างในท่อ/Pan
  • Brix ก่อน Discharge ไม่ถึง → ความเข้มข้นไม่พอ → Purity Drop ต่ำ
โอกาสลด UDL
  • ทำ Hold Vacuum Test ทุก Pan ก่อนเปิดหีบ + ทุกเดือน
  • ควบคุม Boiling Profile ตาม Set Point 10 ขั้น อย่างเคร่งครัด
  • เพิ่ม Purity Drop → ตรวจสอบ B-Mass Brix ≥95, C-Mass Brix ≥97
  • ตรวจ Calandria Tube ก่อนฤดูหีบ (2.5 Kg/cm², 30 นาที)
  • ล้าง Pan หลังใช้งานเพื่อลดน้ำตาลค้าง
  • เริ่มบันทึกปริมาณ Mass/Mol ที่หกล้น/กู้คืนทุกครั้งเป็นตัวเลข เพื่อดึงออกจาก UDL
อ้างอิง WI: ML-WI-3220-001 ถึง 011, ML-SP-3220-001/002/005

ปั่นแยก (Centrifugals)

A/B/C Massecuite → Sugar Crystals + Molasses | แผนก 3230
ขั้นตอนการทำงาน
  1. Massecuite จาก Crystallizer เข้าเครื่องปั่นเหวี่ยง (Batch หรือ Continuous)
  2. ปั่นแยกผลึกน้ำตาลออกจากกากน้ำตาล (Molasses) ด้วยแรงเหวี่ยง
  3. ฉีดน้ำล้าง (Wash Water) ที่ผลึกเพื่อล้างสิ่งเจือปน ปรับปริมาณตามความบริสุทธิ์ของ Massecuite
  4. ตรวจ Purity Rise ของน้ำตาลที่ได้ (เกณฑ์ ≤1.5%) หากไม่ผ่านต้องปั่นซ้ำ
  5. น้ำตาลที่ปั่นได้ส่งเข้าเครื่องอบแห้ง (Dryer) ส่วน Molasses ส่งกลับไปต้มขั้นถัดไป หรือเป็น Final Molasses หากเป็น C-Massecuite
แหล่งที่มาของ Loss
  • Over-washing → ละลายน้ำตาลกลับเข้า Molasses → Molasses Loss + UDL
  • Under-washing → คุณภาพน้ำตาลไม่ดี แต่ Recovery ดี (Trade-off)
  • ตะแกรง (Screen) เสียหาย → ผลึกน้ำตาลหลุดเข้า Molasses
  • น้ำตาลที่ปั่นซ้ำ (Purity Rise ไม่ผ่าน) ไม่ถูกนับเข้า Mass Balance อย่างเป็นระบบ
ความเสี่ยง
  • Screen ฉีก/สึก → น้ำตาลผ่านเข้า Molasses
  • Over-washing ต่อเนื่อง → Molasses Purity สูง → Recovery ลด
  • Basket ไม่สมดุล → การแยกไม่สม่ำเสมอ → ประสิทธิภาพต่ำ
  • ฝุ่นน้ำตาลจากการอบแห้งไม่ถูกเก็บ → Direct Loss
โอกาสลด UDL
  • ปรับ Wash Water ตาม Massecuite Purity — ไม่ใช้ค่าเดียวตลอด
  • ตรวจ Screen ทุกสัปดาห์ เปลี่ยนตาม Schedule
  • ติดตั้งระบบดูดฝุ่นที่มีประสิทธิภาพ + นำน้ำตาลกลับเข้ากระบวนการ
  • กำหนดเป้าหมายเชิงตัวเลขสำหรับปริมาณฝุ่นที่ปล่อยจากการอบแห้ง

การวิเคราะห์คุณภาพ (Quality Analysis)

Lab Analysis ทุกจุด — Cane, Juice, Syrup, Massecuite, Molasses, Bagasse | แผนก 1630
ขั้นตอนการทำงาน
  1. เก็บตัวอย่างจากทุกจุดของกระบวนการ (อ้อย, น้ำอ้อย, น้ำเชื่อม, Massecuite, Molasses, กากอ้อย, กากตะกอนกรอง)
  2. วิเคราะห์ Brix, Pol, Purity ด้วยเครื่อง Polarimeter/Refractometer ตามวิธีที่กำหนดใน WI-1630-001 ถึง 004
  3. คำนวณค่า KPI หลัก (Extraction, Recovery, BHR, UDL) จากผลวิเคราะห์ทั้งหมด
  4. ส่งผลกลับไปแต่ละแผนกเพื่อปรับการควบคุมกระบวนการ (Feedback Loop)
ผลกระทบต่อ UDL
  • ไม่ได้สร้าง Physical Loss แต่ความคลาดเคลื่อนของการวิเคราะห์ส่งผลโดยตรงต่อ "ตัวเลข" UDL
  • วิเคราะห์ Molasses Pol ต่ำกว่าจริง → Molasses Loss ต่ำ → UDL สูงเกินจริง (Phantom UDL)
  • วิเคราะห์ Cane Pol สูงกว่าจริง → Total Input มาก → UDL สูงเกินจริง
ความเสี่ยง
  • Lab Error ใน Cane Pol → UDL ผิดพลาดตลอดทั้ง Season
  • วิเคราะห์ Molasses/Bagasse/Filter Cake ไม่แม่นยำ → Determined Loss คลาดเคลื่อน → UDL เปลี่ยน
  • ความคลาดเคลื่อนสะสม (Cumulative Error) ทำให้ตรวจจับ Real Loss ได้ยาก
โอกาสลด UDL
  • ทำ Cross-check ระหว่างวิธีวิเคราะห์ เช่น Pol vs Direct Polarization
  • ส่งตัวอย่างให้ Lab ภายนอกวิเคราะห์เปรียบเทียบทุกเดือน
  • ลดความคลาดเคลื่อนของ Cane Analysis ด้วยการปรับปรุง Sampling
  • เพิ่มการคำนวณ Purity Drop ระหว่างจุดต่อจุดในทุก Juice Train เพื่อระบุตำแหน่ง Loss จริง

4 สหสัมพันธ์พารามิเตอร์กับ UDL

วิเคราะห์ว่าพารามิเตอร์ใดมีผลต่อ UDL มากที่สุด — ค่า r ยิ่งสูง ยิ่งมีความสัมพันธ์

Correlation with UDL% (r value)

Scatter: Purity Drop vs UDL%

สรุปความสัมพันธ์ที่สำคัญ

Purity Drop ↑ = UDL ↑

Purity Drop สูงสะท้อนว่าการตกผลึกไม่มีประสิทธิภาพ — น้ำตาลหลุดเข้า Molasses มากกว่าปกติ ส่งผลให้เกิด Loss ที่ไม่สามารถระบุได้

Burnt Cane% ↑ = UDL ↑

อ้อยไฟไหม้มี Dextran และ Inversion ของ Sucrose สูง — ทำให้น้ำตาลที่วัดได้จากอ้อยไม่ตรงกับน้ำตาลที่ผลิตได้จริง

Extraction ↑ = UDL ↓

Extraction สูงหมายความว่าการสกัดน้ำตาลจากอ้อยมีประสิทธิภาพสูง — ลดโอกาสที่น้ำตาลจะสูญหายระหว่างกระบวนการ

Capacity ↑ = ต้องระวัง UDL

เมื่อขยายกำลังผลิต (5K→6K→8K t/d) หากกระบวนการไม่พร้อม — Retention Time ลด, ประสิทธิภาพลง → UDL เพิ่ม (เห็นชัดในฤดู 25/26)

Section 05
5 สาเหตุหลักของ UDL & คำแนะนำ

จัดลำดับสาเหตุหลักจากข้อมูลจริง 12 ฤดูหีบ + เอกสาร WI 32 ฉบับ

1

ความคลาดเคลื่อนในการวิเคราะห์ (Analytical Error)

ผลกระทบ: อาจทำให้ UDL สูง/ต่ำกว่าจริง 2-3%

UDL คำนวณจากผลต่างของตัวเลขหลายตัว (Cane Pol, Sugar Produced, Bagasse Pol, Molasses Pol, Filter Cake Pol) — ความคลาดเคลื่อนเล็กน้อยในแต่ละตัวจะ สะสม เข้าด้วยกัน ทำให้ UDL บวมหรือหดได้มาก

  • Calibrate เครื่องมือ: ปฏิบัติตาม ML-WI-3330-002 อย่างเคร่งครัด โดยเฉพาะ Polarimeter, Brix Meter, เครื่องชั่ง
  • Cross-check Lab: ส่งตัวอย่างให้ Lab ภายนอกวิเคราะห์เปรียบเทียบทุกเดือน
  • ปรับปรุง Sampling: ตาม ML-WI-1630-001 ให้สุ่มตัวอย่างเป็นตัวแทนที่ดีขึ้น
  • Daily Sugar Balance: ตรวจสอบ Sugar Balance ทุกวัน ถ้า UDL เกิน 5% → ตรวจ Lab + เครื่องมือทันที
2

การรั่วไหลทางกายภาพ (Physical Leakage)

ผลกระทบ: 1-3% UDL จากการรั่วไหลของ Juice/Syrup/Massecuite

น้ำอ้อย, Syrup, หรือ Massecuite รั่วไหลออกจากระบบ — ลงท่อระบาย, รั่วที่ข้อต่อ, Heater/Evaporator Tube รั่ว — ทั้งหมดเป็น Direct UDL ที่ไม่ถูกวัดในระบบ Loss

  • ตรวจ Brix ใน Condensate: ทุก 2 ชม. — ถ้ามี Brix = Tube รั่ว (Heater/Evaporator)
  • ML-WI-3210-012: ปฏิบัติตามขั้นตอนจัดการ Syrup Spillage อย่างเคร่งครัด
  • ตรวจ Juice Tray ที่ Milling: สัปดาห์ละครั้ง หารอยรั่ว
  • Pressure Test: Calandria Tube ทุก Pan + Evaporator ก่อนเปิดหีบ
  • ติดตั้ง Brix Monitor: ที่ท่อน้ำทิ้ง/Overflow เพื่อตรวจจับการรั่วแบบ Real-time
3

Inversion ของ Sucrose (Sugar Degradation)

ผลกระทบ: 0.5-2% UDL จากการย่อยสลายน้ำตาลซูโครส

Sucrose ถูกย่อยสลายเป็น Glucose + Fructose (Inversion) เมื่อ: pH ต่ำ, อุณหภูมิสูง, เวลานาน, หรือมี Dextran จากอ้อยไฟไหม้ — น้ำตาลที่ถูก Invert แล้วจะไม่ถูกวัดเป็น Pol → หายไปจากระบบ

  • ควบคุม pH: รักษา pH 7.0-7.2 ตลอดกระบวนการ ใช้ Auto Liming
  • ลด Retention Time: ทำความสะอาด Evaporator ตามแผนเพื่อเพิ่มอัตราการระเหย
  • ลดอ้อยไฟไหม้: เจรจากับชาวไร่ ให้สิ่งจูงใจสำหรับอ้อยสด
  • เวลาจากตัดถึงหีบ: ให้สั้นที่สุด โดยเฉพาะอ้อยไฟไหม้ (<24 ชม.)
4

ประสิทธิภาพการตกผลึกต่ำ (Crystallization Inefficiency)

ผลกระทบ: Purity Drop ต่ำ → น้ำตาลค้างใน Molasses → UDL + Mol Loss สูง

  • ควบคุม Boiling Profile: ตาม ML-SP-3220-005 (10-step Set Point) อย่างเคร่งครัด
  • เป้า Brix ก่อน Discharge: B-Mass ≥95, C-Mass ≥97
  • Hold Vacuum Test: ทุก Pan ก่อนเปิดหีบ + ทุกเดือน (ลดลง >20 ใน 20 นาที = รั่ว)
  • Grain Size: ควบคุมตาม Spec — A-Seed 0.8-0.9mm, B-Seed 0.6-0.7mm, C-Seed 0.1-0.2mm
  • Crystallizer Motor: Load <5A — เกินกว่านี้ตรวจสอบทันที
5

Entrainment & ฝุ่นน้ำตาล (Mechanical Losses)

ผลกระทบ: 0.2-1% UDL จาก Entrainment ใน Evaporator + ฝุ่นจาก Drying

  • Entrainment Separator: ตรวจสอบและทำความสะอาดสม่ำเสมอ
  • ระบบดูดฝุ่น: ตาม ML-WI-3230-003 — เก็บฝุ่นน้ำตาลกลับเข้ากระบวนการ
  • Wash Water ที่ Centrifugal: ปรับตาม Massecuite Purity ไม่ใช้ค่าเดียวตลอด
  • Screen Inspection: ตรวจตะแกรง Centrifugal ทุกสัปดาห์
6

ผลกระทบจากการขยายกำลังผลิต (Capacity Expansion Effect)

หลักฐาน: UDL พุ่งจาก 5.10% (24/25, 6K t/d) เป็น 8.02% (25/26, 8K t/d)

  • ทบทวน Retention Time: กำลังผลิตเพิ่ม = อ้อยผ่านแต่ละขั้นเร็วขึ้น → ประสิทธิภาพลด
  • เพิ่ม Capacity ของ Clarifier/Evaporator: ให้สมดุลกับ Crushing Rate ที่เพิ่มขึ้น
  • ตรวจสอบ Pan Boiling Cycle Time: กำลังผลิตสูง → ต้มเร็วขึ้น → อาจไม่ถึงเป้า Brix
  • พนักงานใหม่: ฝึกอบรมให้เข้าใจผลกระทบของแต่ละ Parameter ต่อ UDL

7 แนวคิดโครงการ — จาก Undetermined สู่ Determined

หัวใจของโครงการคือการเปลี่ยนวิธีคิด: UDL ไม่ใช่ "กล่องดำ" แต่เป็น "รายการสาเหตุที่ยังไม่ถูกแยกบัญชี"

Undetermined Loss = Σ (สาเหตุที่ระบุได้ C1–C6) + ส่วนเหลือ (Residual) ที่ต้องบีบให้เล็กลงเรื่อยๆ ทุกฤดู

หลักการที่ 1 — น้ำตาลไม่หายไปเอง

ตามกฎ Mass Balance น้ำตาล (Pol) ที่เข้าโรงงานต้องออกไปทางใดทางหนึ่งเสมอ: เป็นน้ำตาลผลิตภัณฑ์, ติดไปกับชานอ้อย/กากหม้อกรอง/โมลาส, รั่วไหลออกนอกระบบ, ถูกย่อยสลายทางเคมี หรือ "หายไปในกระดาษ" เพราะการวัดผิด — ไม่มีทางเลือกที่ 6

หลักการที่ 2 — แยกให้ออกระหว่าง Loss จริง กับ Loss กระดาษ

UDL มี 2 ชนิดปนกัน: Physical Loss (น้ำตาลหายจริง เช่น รั่วไหล, Inversion, Entrainment) และ Accounting Loss (น้ำตาลไม่ได้หาย แต่ตัวเลขบอกว่าหาย เพราะเครื่องมือวัด/วิธีวิเคราะห์คลาดเคลื่อน) — สองชนิดนี้แก้คนละวิธี จึงต้องแยกก่อน

หลักการที่ 3 — ทุกสมมติฐานต้องพิสูจน์ด้วยการวัด

เราจะไม่เดา เราจะตั้งสมมติฐานจากหลักฐาน (ข้อมูล 12 ฤดู + WI 32 ฉบับ) แล้วออกแบบ "การวัดเพิ่มเติม" เพื่อยืนยันหรือหักล้างทีละข้อ — สาเหตุไหนพิสูจน์ได้ ก็ย้ายออกจาก UDL ไปเป็น Determined Loss ที่มีเจ้าของและมีแผนแก้

8 หลักฐานจากข้อมูล 12 ฤดูหีบ — ทำไมเราถึงเชื่อว่าระบุสาเหตุได้

ก่อนตั้งสมมติฐาน เราดูข้อมูลจริงก่อน ข้อมูลบอกอะไรเราบ้างเกี่ยวกับพฤติกรรมของ UDL

UDL แกว่งแรงเกินกว่าจะเป็น "ธรรมชาติของโรงงาน"

UDL แกว่งจาก 3.97% (23/24) ไป 8.02% (25/26) ภายใน 2 ฤดู — โรงงานเดียวกัน เครื่องจักรส่วนใหญ่เดิม ถ้า UDL เป็นค่าคงที่ตามธรรมชาติ มันไม่ควรแกว่งขนาดนี้ แปลว่ามี "ตัวแปรที่ควบคุมได้" ซ่อนอยู่

UDL เพิ่มพร้อมกำลังผลิต — ชี้ไปที่กระบวนการ ไม่ใช่โชค

ฤดู 25/26 ขยายกำลังผลิตเป็น 8,000 t/d — UDL พุ่งเป็น 8.02%, Purity Drop แย่ลงเป็น 1.022 (แย่ที่สุดตั้งแต่ 14/15), เวลาหยุดเดินเครื่อง (Loss Time) เพิ่มเป็น 5.46% — สามตัวนี้เพิ่มพร้อมกัน ไม่ใช่เรื่องบังเอิญ

สัญญาณ #1 — Trash% = 0 ในฤดู 25/26

ข้อมูลฤดู 25/26 บันทึก Trash% เป็นศูนย์ทั้งฤดู ซึ่งเป็นไปไม่ได้ในทางปฏิบัติ — นี่คือหลักฐานตรงว่า ระบบการวัด/บันทึกข้อมูลมีช่องโหว่ ถ้า Trash วัดผิด น้ำหนักอ้อยสุทธิก็ผิด และ UDL ก็ผิดตาม

สัญญาณ #2 — Purity Drop กับ UDL ไปด้วยกัน

ฤดูที่ Purity Drop สูง (การเคี่ยว/ตกผลึกแย่) UDL มักสูงตาม (r ≈ +0.55) — สอดคล้องกับสมมติฐานว่าส่วนหนึ่งของ UDL คือความสูญเสียจริงในบ้านเคี่ยว (Boiling House) ที่ยังไม่ถูกแยกวัด

สัญญาณ #3 — Downtime สูง = UDL สูง

ฤดู 25/26 Loss Time 5.46% (สูงสุดในรอบ 7 ฤดู) ทุกครั้งที่หยุด-เดินเครื่องใหม่ จะมีน้ำอ้อย/น้ำเชื่อมค้างระบบ ล้างทิ้ง อุณหภูมิ-pH แกว่ง เกิด Inversion — การหยุดบ่อยจึงเป็นแหล่งกำเนิด UDL โดยตรง

สัญญาณ #4 — โรงงานมี WI จัดการน้ำเชื่อมรั่วไหลอยู่แล้ว

การที่มีเอกสาร ML-WI-3210-012 "การจัดการน้ำเชื่อมรั่วไหล" อยู่ในระบบ แปลว่า การรั่วไหลเป็นปัญหาที่เกิดจริงซ้ำๆ จนต้องเขียนวิธีปฏิบัติรองรับ — แต่ยังไม่เคยมีการ "วัดปริมาณ" ว่ารั่วเท่าไหร่ต่อวัน

9 "มันเกิดจากสาเหตุ..." — แจกแจง UDL 8.02% เป็น 6 ก้อน

สมมติฐานการแบ่งสัดส่วน (Loss Attribution Model) จากหลักฐานในข้อมูล 12 ฤดู + WI 32 ฉบับ — ตัวเลขเป็นการประมาณการเบื้องต้นที่ต้องพิสูจน์ในเฟส 1–2 ของโครงการ

สมมติฐานการแจกแจง UDL 8.02% (25/26)

เมื่อพิสูจน์ครบ UDL "ตัวจริง" จะเหลือเท่าไหร่

เป้าหมายไม่ใช่ทำให้ UDL เป็นศูนย์ — แต่คือย้ายแต่ละก้อนออกไปเป็น Loss ที่มีชื่อ มีเจ้าของ มีแผนแก้ จนเหลือ Residual ที่ยอมรับได้ (<1.5%)

หมายเหตุสำคัญ: ตัวเลขประมาณการด้านล่างตั้งจากหลักฐานทางอ้อม (ข้อมูลสถิติ + เอกสาร WI + หลักวิศวกรรมน้ำตาลตามตำรา Peter Rein) ยังไม่ใช่ผลการวัดจริง — ภารกิจของโครงการคือทำให้ตัวเลขเหล่านี้กลายเป็นตัวเลขที่วัดได้จริงภายในฤดูหน้า

10 แผนการวัดเพื่อพิสูจน์ (Measurement Campaign)

"ถ้าวัดไม่ได้ ก็จัดการไม่ได้" — ตารางสรุปว่าแต่ละสมมติฐานต้องวัดอะไร ที่ไหน ถี่แค่ไหน ใครรับผิดชอบ

สาเหตุ สิ่งที่ต้องวัด จุดวัด ความถี่ ผู้รับผิดชอบ เกณฑ์ตัดสิน
C1 การวัดคลาดเคลื่อน ผลวิเคราะห์คู่ (in-house vs external), Calibration record, Trash% Lab 1630, เครื่องชั่งทุกตัว, ลานรับอ้อย รายสัปดาห์ × 4 สัปดาห์ Lab + Instrument (3330) ต่างกัน > ±0.2 Pol = มีปัญหา

C3 Inversion Reducing Sugar Ratio, pH ต่อเนื่อง, เวลาตัด-หีบ Mixed Juice → Syrup แต่ละสถานี รายกะ Lab + Clarification (3210) RS Ratio เพิ่ม > 0.3 = มี Inversion
C4 Entrainment/Condensate Brix/Conductivity คอนเดนเสท, ผล Pressure Test คอนเดนเสททุกหม้อระเหย + หม้อเคี่ยว ทุก 2 ชม. Evaporation + Pan (3220) พบ Brix ใน Condensate = Tube รั่ว
C5 สต๊อกค้างระบบ ระดับ + Brix + Purity ทุก Tank/Pan/Crystallizer บ้านเคี่ยวทั้งหมด สิ้นเดือน + ก่อน/หลังหยุดยาว Pan Boiling (3220) สต๊อกเปลี่ยน > 0.3% Pol = ต้องเข้าสูตร
C6 ฝุ่น/Handling น้ำหนักฝุ่นดักได้, Molasses Purity รายกะ, สภาพ Screen Dryer, หม้อปั่น A/B/C รายวัน / รายสัปดาห์ Centrifugal (3230) Mol Purity สูงขึ้น > 2 จุด = Over-wash/Screen รั่ว

11 Roadmap โครงการ 5 เฟส

จากตั้งสมมติฐาน → วัดจริง → แก้ก้อนใหญ่ → ควบคุมถาวร → UDL กลายเป็นตัวเลขที่อธิบายได้ทุกเดือน

1

เฟส 1 — ตรวจสอบระบบการวัด (Fix the Ruler First)

เดือนที่ 1–2 | เจ้าภาพ: Lab 1630 + Instrument 3330
  • Audit และ Calibrate เครื่องชั่ง เครื่องวิเคราะห์ทุกตัวตาม ML-WI-3330-002
  • Cross-check กับ Lab ภายนอก 4 สัปดาห์ต่อเนื่อง
  • ฟื้นระบบวัด Trash% ให้กลับมาบันทึกจริง
ส่งมอบ: รายงานความคลาดเคลื่อนของระบบวัด + ขนาดของ "UDL กระดาษ" (C1) ที่ยืนยันแล้ว
2

เฟส 2 — แคมเปญวัดทั่วโรงงาน (Measurement Campaign)

เดือนที่ 2–3 | เจ้าภาพ: ทุกแผนกผลิต
  • Loss Walk ทุกกะ, วัด Brix น้ำทิ้ง/คอนเดนเสท, RS Ratio, In-Process Stock Take
  • บันทึกลงแบบฟอร์มกลางแบบเดียวกันทุกแผนก
ส่งมอบ: งบดุลน้ำตาลฉบับแจกแจง — UDL 8.02% ถูกแบ่งเป็น C1–C6 ด้วยตัวเลขวัดจริง
3

เฟส 3 — แก้ก้อนใหญ่ก่อน (Quick Wins ตาม Pareto)

เดือนที่ 3–6 | เจ้าภาพ: แผนกเจ้าของสาเหตุ
  • ซ่อมจุดรั่วตามลำดับปริมาณ, ซ่อม Tube ที่ไม่ผ่าน Pressure Test
  • Auto pH Control, จัดคิวอ้อยไฟไหม้, ปรับ Wash Water ตาม Purity
  • เพิ่มสต๊อกค้างระบบเข้าสูตร Sugar Balance รายเดือน
ส่งมอบ: UDL รายเดือนลดลงอย่างน้อย 2% (จากการแก้ C1+C2 เป็นหลัก)
4

เฟส 4 — ระบบเฝ้าระวังถาวร (Permanent Monitoring)

เดือนที่ 6–9 | เจ้าภาพ: Instrument 3330 + IT
  • ติด Brix/Conductivity Sensor ที่น้ำทิ้งหลักและคอนเดนเสท พร้อม Alarm
  • Dashboard UDL รายวัน แยกตามสาเหตุ C1–C6 ให้ทุกแผนกเห็น
  • ผูก KPI แต่ละสาเหตุกับแผนกเจ้าของ
ส่งมอบ: UDL เปลี่ยนจากรายงานปิดฤดู เป็นตัวเลขรายวันที่ทุกคนเห็นและมีเจ้าของ
